ivar 和 property

前言

长久以来，我一直都有个疑惑，在 MRC 下能否使用 @property 呢？

查阅资料后，我做了一个时间节点：

值得注意的是，在 iOS 上不存在 GC，一直都是引用计数。

很明显了，在 MRC 环境下当然是可以使用 @property 的，只是相对于 ARC 下少了一些修饰词。

MRC 下 property 的修饰词有：

  • atomic/nonatomic
  • assign/retain、copy
  • readonly/readwrite
  • …

ARC 下新增的有：

  • weak/strong

ivar声明

Ivar 可以被声明在类的三个位置：

.h 中：

@interface Foo {
    int ivar1;
    NSNumber *ivar2;
}
@end

extension 中：

@interface Foo () {
    int ivar1;
    NSNumber *ivar2;
}
@end

甚至 .m 中：

@implemention {
    int ivar1;
    NSNumber *ivar2;
}
@end

注意： category 中不能声明 ivar。

访问控制

在 OC 中，ivar 有三个访问控制权限符：

  • protected，只有本类和子类可以访问这个实例变量，这是默认的修饰符。
  • private，只有本类可以访问这个实例变量，extension 和 .m 中声明的 ivar 默认都是 private 的，用其他修饰没有作用。
  • public，外部可以直接访问。

我们一般像这样用：

@interface Foo {
    @private
    int ivar1;
    NSNumber *ivar2;
    @public
    int ivar3;
}
@end

根据面向对象封装的原则，我们让外部尽量少的知道类内部的东西，但是为了让外部能够对 ivar 进行操作，我们一般来说会提供 ivar 的存取方法， getters 和 setters，在 OC 中，默认的 setter 方法是在 ivar 的名称前加 set，默认的 getter 方法直接就是 ivar 的名称，就像下面这样：

首先在 @interface 里面写出方法声明：

@interface Foo {
    int ivar;
}

- (void)setIvar:(int)ivar1;
- (int)ivar;

然后在 @implemention 里面实现方法：

@implemention Foo 

- (void)setIvar:(int)ivar1 {
    ivar = ivar1;
}

- (int)ivar {
    return ivar;
}

@end

这时候你就可以在外部通过这些方法来访问 ivar 了：

Foo *o = [[Foo alloc] init];
[o setA:4];
NSLog(“%d”, [o a]);

OC 还提供了点语法的语法糖供你更方便的调用：

Foo *o = [[Foo alloc] init];
o.a = 4;
NSLog(“%d”, o.a);

property

每次声明实例变量后都要写 getter/setter，这样显得很麻烦，Objective-C 2.0 提供了 @property 语法，由编译器自动为你加上 ivar 和存取方法。

就像下面这样：

@interface Foo
@property (nonatomic, assign) int a;    
@end

@implemtention Foo 
@synthesize a = _a; // Xcode 4.4 and later optional
@end

上面的代码自动的合成一个名为 _a 的实例变量，声明了这个 ivar 的原子性，读写性，内存管理性质，并且为它提供了默认的 getter、setter方法。

在 Xcode 4.4 之后， @synthesize a = _a 会被编译器默认加上，所以这句话也可以省略了。

那么 property 和 synthesize 到底为我们做了什么呢？简单来讲：

property 为 ivar 声明了一些性质，并且提供了 getter/setter 方法。

synthesize 用来合成 ivar，默认一个 property 背后的 ivar 是在 property 的名称前面加上 _ 。

当然你可以重写 getter 和 setter，通常来说，就像下面这样：

- (void)setA:(int)a {
    _a = a + 5; // 对传入的参数做一些操作
}

- (int)a {
    if (!_a) {  // 懒加载，这里的例子可能不合适
        a = 5
    }
    return a;
}

你也可以指定实例变量的名称，但通常不建议这么做：

@synthesize a; // 代表合成的实例变量为 a
@synthesize a = b; // 代表合成的实例变量为 b
