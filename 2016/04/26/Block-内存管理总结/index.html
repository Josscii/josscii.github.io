Block 内存管理总结

前面只是大概的提了一下 block 还不够深入，这里通过回答几个经常碰到的问题来对 block 有个更深入的理解。

名词解释：

  • automatic variable、local variable、局部变量都代表一个东西，存活在栈上。

  • block literal (字面量)，例：^ { printf(“hello world\n”); }

问题1

通常来说，我们会把 block literal 赋值给声明的 block 变量，那么这些 block 都是什么类型呢？

以下规则适用于 MRC & ARC：

当block literal 被赋值给函数内部的局部变量时，一开始会被创建在栈上， isa 指针指向 _NSConcreteStackBlock,
当block literal 被赋值给 static 的变量或者全局变量时，会被创建在 text 区，isa 指针指向 _NSConcreteGlobalBlock，
特别的，当block 没有引用外部变量时，clang 会做一个优化，这时的 block 为 _NSConcreteGlobalBlock。

如果把 block 进行 copy （block_copy）操作后，该 block 会被 copy 到堆上，变成 _NSConcreteMallocBlock。

特别的，由于在 ARC 中对象声明都默认有 __strong 修饰，所以此时的 block 会被 copy 到堆上，这点需要注意。

问题2

我们都知道 block 会捕获外部的变量，那么捕获变量又有哪些规则呢？

以下规则适用于 MRC & ARC：
对于局部的基本数据类型的变量，block 会 copy 一份值，并且不可修改。
对于全局、静态、属性基本类型变量，blcok 和外部共享一份，可以修改。

有区别的地方：

对于局部的对象，non-arc 会copy一份指针，并且不会 retain；arc 会强引用（retain，引用计数加一），均不可修改。

对于属性对象 & self，arc下会被 retain，
而 non-arc 需要分情况：
如果为 stack block，那么就不会被 retain；
如果为 malloc block，就会被 retain。

(注：除了 NSString *可以声明为静态、全局的对象外，其他的对象都不行，字符串比较特殊，可以作为编译时常量)

问题3

如果我们修改捕获的变量，又该怎么办呢？

我们可以在 block 声明的前面加上 __block 修饰，此时，就可以修改被捕获的变量了，并且会影响外部的值。

只是要注意的是，只要 block 被 copy 后（无论是被强引用还是手动 copy），捕获的变量都会被 copy 到堆上，此时原来栈上的 __block 变量会被销毁。

问题4

什么时候会产生循环引用？如何解决呢？

简单来说，只要 self 持有了 block，而在 block 里又对 self 进行了捕获，那么就会导致循环引用，所以一般的栈上的 block 就不会引起循环引用。

解决方案就是打破循环引用，一般来说：
在 MRC 下，创建一个用 block 修饰的指向 self 的指针，在 block 里使用它。
在 ARC 下，创建一个用 weak 修饰的指向 self 的指针，在 block 里使用它。
